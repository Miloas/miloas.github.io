JS == 运算符

发表于 2016-10-25   |  

偶然看到司徒正美前辈的2014年的一篇博客javascript 隐性类型转换步骤，看到评论里有人对练习的最后一个有疑问. 其实这里确实是正美前辈没说清楚..

为什么这个是true呢？

javascript的类型有两类Primitive和Object，Primitive有Undefined,null,Boolean,Number和String.

先说undefined,null和NaN，NaN不等于任何值（包括它自身），null和undefined相等，但不等于除了自身外的任何值.

（ps: null的含义是空对象, undefined的含义是无定义的值)

除了和对象的比较，其他的比较都是大家都转数字来进行比较.

对象和非对象的比较的规则稍微复杂一点，不过也只是一点点. 过程就是先valueOf一下，看有没有得到primitive的值或者null，如果得到了就直接比较，否则就去试toString，看看返回的值是不是primitive和null，是的话就比较，不是的话抛异常.

回到一开始的问题{ toString:function(){ return 1 } , valueOf:function(){ return [] }}调用valueOf()返回的是[]不是我们需要的可比较类型，所以再用原object调用toString()返回1，直接比较.
